# Example Project: NSDI Tail-Latency Paper

This example shows what a `researchstack`-managed project can look like once the pack has been used for a while.

It is not meant to be executable code.

It is meant to show the shape of the working artifacts:

- paper brief
- project memory
- experiment matrix
- figure storyboard
- reviewer-style feedback
- rebuttal plan

Suggested order:

1. Read `brief.md`
2. Read `project-memory.memory.jsonl`
3. Read `experiment-matrix.md`
4. Read `figure-storyboard.md`
5. Read `peer-review.md`
6. Read `rebuttal-plan.md`
